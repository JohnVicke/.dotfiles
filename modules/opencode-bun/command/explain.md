Explain the specified code/error/concept.

## Behavior

1. **Assume programming knowledge**: Don't explain basic concepts
2. **Be concise**: Bullet points preferred, max 10 lines unless asked for more
3. **Include gotchas**: Common mistakes and pitfalls
4. **Link docs if relevant**: Official documentation for further reading

## Format

```
## What it does
- Point 1
- Point 2

## Gotchas
- Common mistake 1
- Pitfall to avoid

## See also
- Link to docs
```

## Rules

- **Concise over comprehensive** - user can ask for more detail
- **Practical over theoretical** - focus on usage, not history
- **Specific over generic** - answer the actual question, not related topics
